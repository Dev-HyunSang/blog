안녕하세요. JavaScript Engine 작동 원리와 자바스크립트 컴파일 방식에 대해 궁금증이 생겨서 작성해 보고자 합니다.  
Rust로 개발한 자바스크립트 엔진인 [`boa`](https://github.com/boa-dev/boa)에서 궁금증이 생겨서 작성합니다.  

## 자바스크립트 엔진이란?
자바스크르립트 코드를 실행하는 프로그램 또는 인터프리터입니다. 최초의 엔진은 단순한 인터프리터였지만, 모든 현대적인 엔진은 JIT(Jut-In-Time Compliation) 또는 런타임 컴파일을 사용하여 성능을 향상시키고 있습니다.  

### JIT(Just-In-Time Compliation) 방식은?
JIT(Just-In-Time Compliation) 또는 동적 번역(Dynamic Translation)은 프로그램을 실제 실행하는 시점에 기계어로 번역하는 컴파일 기법을 말합니다.  
JIT 컴파일러는 실행 시점에서 인터프리트 방식으로 기계어 코드를 생성하면서 캐싱(메모리에 데이터를 임시 저장 후 불러쓰는 것)합니다. 그렇기 때문에 함수가 여러 번 불릴 때 매번 기계어 코드를 생성하는 것을 방지할 수 있습니다.  
이런 측면에서 JIT 컴파일러는 인터프리트 방식과 정적 컴파일 방식을 혼합한 방식으로 볼 수 있습니다.
